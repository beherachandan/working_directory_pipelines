# Persona: Assessment Director / Data Leader

## Role & Title
- **Title:** Director of Assessment, Director of Data & Accountability, Assessment Coordinator, Data Leader, Director of Research & Evaluation
- **Level:** District-level administrator — critical for CA deals
- **Scope:** Assessment strategy, testing calendar, benchmark programs, data aggregation, state test prep, accountability metrics, reporting to board/state
- **Nickname:** "The Evidence Gatekeeper"

**Why This Persona Matters:**
This persona determines whether Wayground's data is "valid enough" to take seriously as a system. They control whether Wayground is viewed as an engagement toy or a serious assessment platform. For CA deals, they are often the primary decision maker or key influencer. They own the assessment budget and testing calendar.

## Primary Concerns
1. **Assessment quality & integrity** — psychometric soundness, test security, lockdown capability
2. **Data accuracy & aggregation** — consistent, cross-school data they can trust and report on
3. **Standards mastery tracking** — growth over time by standard, subgroup, school
4. **State test correlation** — do our internal assessments predict state test performance?
5. **Reporting to board & state** — clean dashboards, accountability metrics, gap analysis
6. **Common Assessment coordination** — standardized assessments across buildings, consistent administration

## Language They Use
- "We don't trust our current data"
- "Teacher data lives in too many places"
- "We can't see progress by standard"
- "Our benchmark data doesn't predict state test performance"
- "We need item-level analysis, not just total scores"
- "I need data by subgroup — race, ELL, SPED, economically disadvantaged"
- "How does this compare to what we get from [Mastery Connect / Performance Matters / Illuminate]?"
- "We need cross-school comparison on the same assessment"

## Decision Authority
- **Can approve:** Assessment tool evaluations, testing calendar decisions, benchmark program selection
- **Can recommend:** District-wide CA platform adoption to superintendent
- **Can block:** If they don't trust the data, the deal dies — even if teachers love it
- **Delegates to:** Assessment coordinators at building level, PLC leads for item analysis
- **Influenced by:** State accountability requirements, psychometric standards, peer district practices

## Objection Patterns
- "Your assessments aren't rigorous enough — this is a game platform"
- "We need item banks with psychometric validation"
- "How do you handle test security? Teachers can share answers"
- "Our current tool [PM/MC/Illuminate] gives us deeper analytics"
- "We need bubble sheet scanning for paper tests"
- "Can you import our existing item banks?"
- "The data needs to integrate with our SIS/LMS"

## Win Triggers
- **Legacy tool pain:** Current CA platform is aging, unreliable, or vendor got acquired (Performance Matters → PowerSchool decay, Mastery Connect → Instructure issues, Illuminate → HMH neglect)
- **CA-like behavior:** Teachers already running same quiz across sections/schools organically — formalizing with CA is natural
- **Failed CA adoption:** Previous CA tool had low teacher adoption because teachers hated it
- **Data fragmentation:** Assessment data scattered across 5+ platforms
- **State test alignment:** Need assessments that mirror state test format and rigor
- **PLC data needs:** Strong PLC culture that needs common assessment data to drive conversations

## Data Points That Matter
Lead with assessment and data metrics:
- **CA-like behavior** — teachers already coordinating assessments across sections/buildings
- **Assessment volume** — total assessments created/administered, question types used
- **Standards coverage** — which standards are being assessed, gaps in coverage
- **Cross-school consistency** — same quiz used by multiple teachers in same org/district
- **Data quality indicators** — item analysis, discrimination index, reliability metrics
- **Competitive tool presence** — what assessment platform they currently use (from SF data)

## How to Engage
- **Lead with data credibility** — they need to trust the assessment engine before anything else
- Show lockdown browser, test security, anti-cheating features first
- Demonstrate cross-school analytics: same assessment, multiple buildings, one dashboard
- Highlight item-level analysis, not just total scores
- Reference state test alignment: question types that mirror their state assessment
- If they use a legacy tool (PM, MC, Illuminate), acknowledge its strengths but lead with teacher adoption: "The best assessment data comes from a tool teachers actually use"
- Show that 48.2% of districts have NO mandated CA tool — this is greenfield, not displacement

## CA-Specific Positioning
- **The killer narrative:** "Your district already has 47 teachers running the same unit test across 8 schools every month. They're doing Common Assessments without calling it that. CA just gives you the admin dashboard, lockdown browser, and cross-school analytics to make it official."
- **Against Mastery Connect:** "Deep standards tracking, but reliability issues post-Instructure acquisition. Teachers don't use it daily — adoption friction kills data quality."
- **Against Performance Matters:** "Comprehensive benchmarking, but terrible authoring (only 16% of teachers create assessments in it). Heavy admin setup."
- **Against Illuminate/DnA:** "Strong MTSS workflows, but complex, expensive, and teachers don't engage daily. 'Lots of technical issues, lack of support.'"
- **Wayground advantage:** Already in the building. Teachers use it every day. CA is an upgrade, not a new tool.

- **When they say:** "We don't trust our current data" → Lead with MEASURE
- **When they say:** "We need common assessments" → Lead with MEASURE + CA positioning
